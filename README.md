# SQL Learning Journal

A public record of my SQL interview preparation, data science & analytics problem-solving, and database learning journey.

## Purpose

This repository serves as a structured SQL learning journal where I document:

* SQL interview practice
* Analytics case studies
* Common mistakes and debugging experiences
* SQL concepts and mental models
* Window functions
* Aggregations
* Joins
* CTEs
* Query optimization techniques

The goal is not simply to collect solutions.

The goal is to build a deep understanding of SQL and maintain a transparent record of my learning process over time.

* How I approached a problem

This repository functions as both:

## Learning Philosophy

For every problem solved, I aim to answer:

1. What was the business question?
2. What was my first instinct?
3. What mistakes did I make?
4. Why did those mistakes happen?
5. What SQL concepts were involved?
6. What would I do differently next time?

The objective is understanding, not memorization.
